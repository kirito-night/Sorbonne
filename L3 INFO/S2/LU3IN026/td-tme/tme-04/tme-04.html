def plot_frontiere_V3(desc_set, label_set, w, kernel, step=30, forme=1, fname="out/tmp.pdf"):
    """ desc_set * label_set * array * function * int * int * str -> NoneType
        Note: le classifieur linaire est donn sous la forme d'un vecteur de poids pour plus de flexibilit
    """
    # -----------
    # ETAPE 1: construction d'une grille de points sur tout l'espace dfini par les points du jeu de donnes
    mmax=desc_set.max(0)
    mmin=desc_set.min(0)
    x1grid,x2grid=np.meshgrid(np.linspace(mmin[0],mmax[0],step),np.linspace(mmin[1],mmax[1],step))
    grid=np.hstack((x1grid.reshape(x1grid.size,1),x2grid.reshape(x2grid.size,1)))
    
    # -----------
    # Si vous avez du mal  saisir le concept de la grille, dcommentez ci-dessous
    #plt.figure()
    #plt.scatter(grid[:,0],grid[:,1])
    #if True:
    #    return
    
    # -----------
    # ETAPE 2: calcul de la prediction pour chaque point de la grille
    res=np.array([kernel(grid[i,:])@w for i in range(len(grid)) ])
    # pour les affichages avancs, chaque dimension est prsente sous la forme d'une matrice
    res=res.reshape(x1grid.shape) 
    
    # -----------
    # ETAPE 3: le trac
    #
    # CHOIX A TESTER en dcommentant:
    # 1. lignes de contours + niveaux
    if forme <= 2 :
        fig, ax = plt.subplots() # pour 1 et 2
        ax.set_xlabel('X_1')
        ax.set_ylabel('X_2')
    if forme == 1:
        CS = ax.contour(x1grid,x2grid,res)
        ax.clabel(CS, inline=1, fontsize=10)
    #
    # 2. lignes de contour 0 = frontire 
    if forme == 2:
        CS = ax.contour(x1grid,x2grid,res, levels=[0], colors='k')
    #
    # 3. fonction de dcision 3D
    if forme == 3 or forme == 4:
        fig = plt.gcf()
        ax = fig.gca(projection='3d') # pour 3 et 4
        ax.set_xlabel('X_1')
        ax.set_ylabel('X_2')
        ax.set_zlabel('f(X)')
    # 
    if forme == 3:
        surf = ax.plot_surface(x1grid,x2grid,res, cmap=cm.coolwarm)
    #
    # 4. fonction de dcision 3D contour grid + transparence
    if forme == 4:
        norm = plt.Normalize(res.min(), res.max())
        colors = cm.coolwarm(norm(res))
        rcount, ccount, _ = colors.shape
        surf = ax.plot_surface(x1grid,x2grid,res, rcount=rcount, ccount=ccount, facecolors=colors, shade=False)
        surf.set_facecolor((0,0,0,0))
    
    # -----------
    # ETAPE 4: ajout des points
    negatifs = desc_set[label_set == -1]     # Ensemble des exemples de classe -1
    positifs = desc_set[label_set == +1]     # +1 
    #Affichage de l'ensemble des exemples en 2D:
    if forme <= 2:
        ax.scatter(negatifs[:,0],negatifs[:,1], marker='o', c='b') # 'o' pour la classe -1
        ax.scatter(positifs[:,0],positifs[:,1], marker='x', c='r') # 'x' pour la classe +1
    else:
        # on peut ajouter une 3me dimension si on veut pour 3 et 4
        ax.scatter(negatifs[:,0],negatifs[:,1], -1, marker='o', c='b') # 'o' pour la classe -1
        ax.scatter(positifs[:,0],positifs[:,1], 1,  marker='x', c='r') # 'x' pour la classe +1
    
    # -----------
    # ETAPE 5 en 3D: rgler le point de vue camra:
    if forme == 3 or forme == 4:
        ax.view_init(20, 70) # a rgler en fonction des donnes
    
    # -----------
    # ETAPE 6: sauvegarde (le nom du fichier a t fourni en argument)
    if fname != None:
        # avec les options pour rduires les marges et mettre le fond transprent
        plt.savefig(fname,bbox_inches='tight', transparent=True,pad_inches=0)

Compléments sur le perceptron¶

Afin d'amliorer notre classe ClassifierPerceptron, nous allons considrer une version modifie de cette classe, que nous appellerons ClassifierPerceptronBias et qui construira un perceptron avec:

  • un biais qui facilite la convergence du modle: le critre de mauvaise classification est remplac par $$ f(\mathbf x_i) y_i < 1 $$ o $f(\mathbf x_i)$ est le score obtenu pour $x_i$.

Ide: on veut que $f(\mathbf x_i)$ et $y_i$ soient du mme signe ET que $f(\mathbf x_i)$ soit suffisamment grand (en valeur absolue) C'est une version stabilise du perceptron o les points ne sont considrs bien classs que lorsque $f(\mathbf x_i) y_i>1$. Dans le cas contraire, on met jour les poids.

  • la mise jour des poids tient compte de ce biais: $$ w = w +\epsilon (y_i- f(\mathbf x_i))x $$

Cette nouvelle classe doit contenir :

  • un attribut allw (par exemple, de type liste) pour stocker toutes les valeurs successives de w rencontre lors de l'entranement du perceptron. Cet attribut est donc mettre jour lors de l'entranement du perceptron, aprs chaque mise jour des poids. Remarque: attention ! pour copier le vecteur $w$ penser faire une copie profonde...
  • un accesseur get_allw() permettant de rcuprer la valeur de allw

Nous avons vu en cours que l'volution des poids du perceptron correspond en ralit l'optimisation de la fonction cot:

$$ \mathcal C = \sum_{i=1}^N (1- f(\mathbf x_i) y_i)_+, \qquad \mbox{avec: } (\alpha)_+ = \left\{\begin{array}{ll} \alpha & \mbox{ si } \alpha >0\\ 0 & \mbox{ sinon } \end{array}\right.$$

Nous utilisons ici une version stabilise du perceptron o les points ne sont bien class que lorsque $f(\mathbf x_i) y_i>1$

Le perceptron est une simple descente de gradient.

Validation Croisée¶

La procdure de la validation croise est au centre de la plupart des applications de machine learning: il est temps pour nous de travailler sur une implmentation.

L'ide est de concevoir la fonction suivante:

Xapp,Yapp,Xtest,Ytest = crossval(X, Y, n_iterations, iteration)
  • X, Y sont les donnes du dataset mlanges alatoirement
  • n_iterations est le nombre d'ensembles de test au total.
  • iteration est l'itration concerne: on ne renvoit pas les mmes donnes en fonction des itrations.

Aprs sparation des indices en n_iterations groupes, isoler 1 groupe pour le test et les autres pour l'apprentissage.

Version multi-classes¶

Le perceptron est un algorithme fondamentalement bi-classes: il est mme li la manire dont sont encodes les tiquettes $\{+1,-1\}$. La plupart des problmes rels sont pourtant multi-classes (e.g. USPS). Si cet algorithme a pu rester aussi classique, c'est qu'il existe une extension multi-classes simple et efficace: le un-contre-tous

Il faut apprendre autant de classifieurs que de classes. Dans l'exemple ci-dessus, cela donne:

  1. Traitement de la classe 0
  2. redfinition de: $$Y_{tmp} = \left\{ \begin{array}{ll} 1 & \text{ si } Y == 0\\ -1 & \text{ si } Y == 1 || Y == 1\\ \end{array} \right. $$
  3. apprentissage de $f_0$ sur $X,Y_{tmp}$
  4. Traitement de la classe 1
  5. redfinition de: $$Y_{tmp} = \left\{ \begin{array}{ll} 1 & \text{ si } Y == 1\\ -1 & \text{ si } Y == 0 || Y == 2\\ \end{array}\right. $$
  6. apprentissage de $f_1$ sur $X,Y_{tmp}$
  7. etc...

En infrence, lorsqu'un nouveau point $\mathbf x$ est prsent il faut:

  1. calculer $score = [f_0(\mathbf x), f_1(\mathbf x), \ldots f_C(\mathbf x)]$
  2. prdire $\text{argmax}_c (score)$

Plus loin avec le multi-classes¶

On ne va pas dfinir une version multi-classe par classifieur binaire... Il est possible de dfinir un classifieur gnrique multi-classe partir de classifieur binaire (design pattern composite) en respectant le schma suivant:

  1. ClassifierMultiOAA prend un classifieur binaire en argument lors de la cration. Il initialise une liste vide de classifieurs.

  2. Dans la mthode train, penser cloner le classifeur de rfrence nCl fois. Il faut utiliser la mthode deepcopy du module standard python copy

  3. Dans un boucle for, redfinir les tiquette $ytmp$ et apprendre des classifieurs binaires

  4. Dans score faire appel aux mthodes score des classifieurs binaires et stocker les rsultats

  5. Dans predict renvoyer l'argmax des scores.
    Note: en faisant cette opration, on fait l'hypothse que les tiquettes sont dfinies sur $[0,nCl]$. Pour lever cette hypothse, il faudrait stocker les classes et convertir le rsultat de argmax.

  6. La fonction accuracy de la classe mre doit tre compatible avec les dfinitions prcdentes... Par exemple:

    def accuracy(self, desc_set, label_set):
     yhat = np.array([self.predict(x) for x in desc_set])
     return np.where(label_set == yhat, 1., 0.).mean()
